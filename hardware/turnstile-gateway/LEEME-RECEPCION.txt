LECTOR DE TARJETA - PC DE RECEPCION (Windows)
=============================================

IMPORTANTE: copie TODA esta carpeta al PC. No sirve copiar solo el .bat.

Archivos obligatorios (misma carpeta):
  - serial_card_reader.py
  - iniciar-lector-tarjeta.bat
  - detener-lector-tarjeta.bat

Como usar:
  1. Conecte el lector USB (COM3 en Administrador de dispositivos).
  2. Doble clic en iniciar-lector-tarjeta.bat
  3. Debe verse: "Escuchando COM3 @ 9600 baud"
  4. Pase una tarjeta -> sale el numero en pantalla.

Si sale "No such file or directory":
  -> Ejecute descargar-lector-recepcion.bat (con internet)
  -> O copie de nuevo toda la carpeta con serial_card_reader.py

Formato por defecto: HEX (UID del chip, ej. A5AD8AE2), como el sistema anterior.
  -> iniciar-lector-tarjeta.bat ya trae SERIAL_PIN_FORMAT=hex
  -> Vincule en el gym el mismo codigo que muestra al pasar la tarjeta

Si el diagnostico muestra RECIBIDO en 9600 (ej. hex A5AD8AE2):
  -> Cierre ZKAccess, use iniciar-lector-tarjeta.bat
  -> El script envia el UID en hex automaticamente

Teclado del lector ZKT: si al pulsar un numero sale una letra:
  -> En el terminal ZKT: usuario/tarjeta en modo NUMERICO (no alfanumerico).
  -> En el PC (campo web): use teclado numerico o pase la tarjeta.

Solo si ZKAccess muestra solo numeros (sin letras A-F), cambie en iniciar-lector-tarjeta.bat:
     set SERIAL_PIN_FORMAT=decimal

Si pasa tarjeta y NO sale nada en pantalla:
  -> probar-diagnostico-com.bat (lista COM y prueba 9600/115200)

Si el torniquete quedo LIBRE (sin seguro) y no vuelve a bloquearse:
  -> Doble clic en poner-seguro.bat (misma carpeta turnstile-gateway)
  -> Cierre quitar-seguro.bat si lo dejo abierto
  -> Reinicie iniciar-lector-tarjeta.bat o Sport Gym Acceso
  -> El script ahora re-pone seguro solo tras 8 s; si falla COM3, use poner-seguro.bat

INSTALACION OFICIAL PC DEL TORNIQUETE (app .exe + lector + acceso directo):
  -> En el PC del torniquete ejecute SOLO:
     ACTUALIZAR-TORNIQUETE-DESDE-GIT.bat
     (copielo desde hardware/ del repo o descargue de GitHub)
  -> Ver hardware/LEEME-TORNIQUETE.txt
  -> Instala Sport Gym Acceso.exe + lector + icono gym en escritorio

APP .EXE (solo quien compila en desarrollo):
  -> hardware/COMPILAR-Y-PREPARAR-INSTALADOR.bat
  -> hardware/SUBIR-SETUP-AL-SERVIDOR.bat
  -> Luego en torniquete: ACTUALIZAR-TORNIQUETE-DESDE-GIT.bat

APP sin Electron (emergencia):
  -> SportGym-Acceso-App.bat     ventana tipo app + lector
  -> SportGym-Acceso-Kiosk.bat   pantalla completa + lector

INICIO AUTOMATICO (al encender Windows):
  -> instalar-inicio-automatico.bat (usa SportGym-Acceso-App.bat)
  -> O cada dia: iniciar-puesto-acceso.bat

La pagina web sola NO ejecuta el .bat del lector; los .bat de esta carpeta si.
El lector debe estar en el MISMO PC que tiene COM3, o siempre encendido con iniciar-lector-tarjeta.bat.

PLACAS (ver ARQUITECTURA-PLACAS.txt):
  AZUL = parte del PC (pantalla, audio, ethernet). NO controla el torniquete.
  CAFE = torniquete (lector pared, relé, seguro).

SIN MANUAL — como encontrar control del seguro:
  1. Placa CAFE: ¿cable de red al switch? ¿otro cable al PC ademas del USB lector?
  2. Si la cafe tiene IP en la red del gym -> probar TURNSTILE_GATE_MODE=http en .env
  3. Si hay otro COM (no COM3 lector) hacia la CAFE -> escuchar-puerta-com.bat
  4. Si solo lector de pared -> el seguro lo mueve la cafe; Sport Gym solo valida en pantalla

CONFIG LISTA (turnstile-gate.env en esta carpeta):
  BLOQUEAR: d (o b,c,e)  LIBERAR: a  COM3 @ 19200
  Cierre ATP-ACCESO 4.0.exe antes de Sport Gym.

PUERTA FISICA (seguro — placa CAFE, no la azul):
  El torniquete NO gira solo: quita el SEGURO para empujar.

  turnstile-gate.env (copie .example):
    serial  -> COM hacia placa CAFE si existe (NO placa azul, NO COM3 lector)
    http    -> URL si la placa CAFE responde por red (IP local)
    none    -> solo pantalla/API; seguro lo maneja la cafe o ZKAccess

  Comportamiento deseado con .env bien configurado:
    - Al abrir Sport Gym Acceso: PONE seguro.
    - GRANTED: quita seguro unos segundos, vuelve a poner.
    - Denegado: mantiene seguro.

  Pruebas: poner-seguro.bat / quitar-seguro.bat

  VPS: TURNSTILE_WEBHOOK=http://IP_PLACA_CAFE:PUERTO/ruta (si la cafe tiene API en red)

Flujo dos PC (recepcion + entrada):
  PC entrada: .exe o iniciar-lector-tarjeta.bat + SportGym-Acceso
  PC recepcion: sportgymr10.com -> Acceso -> INGRESOS (abierto todo el dia)
  Cliente "quiero entreno" -> en entrada pulsa F2 (o tarjeta) -> usted ve la fila en Ingresos
  Ver FLUJO-DOS-PC.txt en access-desktop/

Apertura remota en LAN (recepcion -> torniquete):
  1) En PC torniquete debe estar abierto iniciar-lector-tarjeta.bat (puerto 8765 activo).
  2) En PC recepcion ejecute:
     - abrir-puerta-remota.bat   (abre)
     - cerrar-puerta-remota.bat  (pone seguro)
  3) Si cambia la IP del torniquete, edite TARGET_IP en esos .bat.

Flujo completo:
  Tarjeta -> script (PC entrada) -> API -> GRANTED -> TURNSTILE_WEBHOOK -> puerta
                              \-> /acceso bienvenida + Ingresos en recepcion (polling)

Para liberar COM3 (ZKAccess u otra app):
  -> detener-lector-tarjeta.bat
